Claude Code 2.1.80 — Channels let you message your coding agent from Telegram

Yesterday Claude Code Changelog / VentureBeat

Claude Code 2.1.80 introduces Channels (research preview): MCP servers that push messages from external platforms into active coding sessions. Launch platforms are Telegram and Discord with two-way support — Claude reads the incoming event and can reply back through the same channel. The feature requires the Bun runtime and uses the --channels flag. Also in this release: 80MB memory savings on 250K-file repos, rate limit display in statusline scripts, effort frontmatter support for skills, and a fix for --resume dropping parallel tool results.

VentureBeat frames Channels as an “OpenClaw killer” — instead of building a standalone agent bot, Anthropic made your existing local session (with full filesystem access and context) reachable from any messaging app. The session runs on your machine; Telegram becomes the remote control. For anyone who kicks off long agent runs and walks away, this is the feature you were waiting for.

Anthropic’s 80,000-person study: people fear AI unreliability more than job loss

Yesterday TLDR AI / Anthropic

Anthropic published results from a global study of 80,508 participants on public attitudes toward AI — one of the largest surveys of its kind. Top desires: using AI for professional excellence and personal improvement. The leading concern wasn’t job displacement — it was AI unreliability. Respondents want AI that works consistently before they want it to do more. The study surveyed participants across demographics and geographies, covering both current use patterns and forward-looking expectations.

TLDR AI’s take: at 80K participants, this dwarfs most AI sentiment research. The finding that “unreliability” outranks “job loss” as a concern suggests the public is more pragmatic than the pundit class — people who actually use AI daily care more about it working well than about existential labor-market fears.

Karpathy: humans are now the bottleneck in AI research

Today WinBuzzer / GitHub

Karpathy’s open-source autoresearch framework ran 700 autonomous experiments over two days on a single GPU, finding ~20 additive optimizations that yielded an 11% training speed gain. After the first night alone, the agent had run 50 experiments, discovered a better learning rate, and committed the proof to git — zero human input. Shopify CEO Tobias Lütke ran 37 overnight experiments on internal data and reported a 19% performance gain. Karpathy’s next goal: asynchronous swarms of agents collaborating across scales, emulating “a research community” rather than a single PhD student.

Karpathy’s argument: in any domain with a clear, computable metric, AI agents already find optimizations humans miss. The bottleneck isn’t compute or models — it’s the speed at which humans can formulate and test hypotheses. Alignment and interpretability, where metrics are ambiguous, remain firmly human territory.

Claude Code 2.1.81: --bare scripting flag and --channels permission relay

Mar 20 Anthropic changelog

Claude Code v2.1.81 shipped two notable features. The --bare flag strips hooks, LSP, plugin sync, and skill walks from scripted -p calls — purpose-built for CI pipelines and automation. The --channels permission relay (research preview) lets channel servers forward tool-approval prompts to your phone via Telegram or Discord, so you can approve agent actions from anywhere. Also fixed: OAuth token refresh across concurrent sessions, voice mode silent retries, and session resumption in worktrees.

The --bare flag is Anthropic acknowledging that Claude Code is becoming infrastructure, not just a dev tool. Stripping startup overhead for scripted calls means they’re optimizing for the “fleet of agents” pattern that CNBC just reported developers are adopting.

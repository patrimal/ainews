AI vulnerability tools are drowning open-source maintainers — cURL’s 181-report year

Today Bloomberg / The New Stack

Bloomberg reports that AI tools like Mythos are finding security flaws far faster than small volunteer teams can address them. Daniel Stenberg, creator of cURL, received 181 bug and vulnerability reports in 2025 alone — “roughly as many as the previous two years combined.” His seven-person volunteer team found and fixed more vulnerabilities in Q1 2026 than in each of the prior two full years. The quality has improved too — roughly 1 in 10 reports are actual security vulnerabilities, with “almost all the bad ones” gone. But the volume is unsustainable for the underfunded teams that maintain critical internet infrastructure.

Stenberg’s warning: “It’s an overload of all the maintainers who are already often overloaded and understaffed and underpaid and underfunded.” AI is finding real bugs in code that survived decades of human review — but the humans who fix those bugs are volunteers.

Claude Code v2.1.113 ships — native binary, sandbox domain deny-lists, /loop gets Esc-cancel

Today Anthropic changelog

Anthropic pushed v2.1.113 today with what may be the biggest structural change Claude Code has had in months: the CLI now spawns a native platform binary via optional per-platform dependencies instead of bundled JavaScript. The release adds sandbox.network.deniedDomains for explicit domain blocklists, macOS path protection for /private/{etc,var,tmp,home}, and Bash deny rules that now match wrapped commands. Fullscreen mode gains Shift+↑/↓ selection scrolling and proper Ctrl+A/E line navigation. /loop can now be canceled with Esc, and /ultrareview runs checks in parallel. Long URLs stay clickable when wrapped across lines.

The native-binary move is the quiet headline. Bundled-JS startup has been a persistent pain point for anyone running Claude Code in CI or in tight container environments — cold start and memory footprint both shrink when the CLI is actually compiled per-platform. The deniedDomains and wrapped-command deny rules close two common sandbox-escape paths that security-minded users have been complaining about.

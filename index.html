Karpathy proposes “LLM Wiki” — a living knowledge layer that makes the case RAG is broken

Apr 9 Andrej Karpathy / GitHub

Karpathy published a GitHub Gist outlining “LLM Wiki” — a pattern where an LLM maintains a persistent, interlinked markdown knowledge base instead of retrieving raw documents on every query. When new sources arrive, the LLM summarizes them, links to existing articles, and writes updates back into the wiki. He praised “Farzapedia” as a real implementation: 2,500 personal entries compiled into 400 detailed, backlinked articles. His core argument: RAG never accumulates knowledge — every complex query requiring synthesis across multiple documents starts from scratch.

Karpathy’s argument: the knowledge artifact is explicit, navigable, and local — not implicit in some provider’s opaque memory system. You can see exactly what the AI does and doesn’t know. Multiple analyses are calling this a paradigm shift for personal knowledge management, replacing RAG’s “search every time” approach with structured, accumulated understanding.

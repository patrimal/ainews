Karpathy: “I’ve never felt this behind as a programmer” — agents now write 80% of his code

Apr 7 Karpathy on X / Futurum / Medium

Andrej Karpathy posted that the programming profession is being “dramatically refactored” as the bits contributed by human programmers become “increasingly sparse and between.” He went from writing 80% of his own code to almost none in a matter of weeks, calling it “easily the biggest change in ~2 decades of programming.” He now runs multiple coding agents in parallel on a tiled monitor, assigning features to each and reviewing outputs as they return. His takeaway: developers can be “10X more powerful” if they learn to orchestrate what’s now available — and a failure to do so is “decidedly a skill issue.”

Futurum’s analysis: the speed of the transition — weeks, not months — suggests a capability discontinuity rather than gradual adoption. Karpathy’s point isn’t that AI writes better code; it’s that the developer role has shifted from “writing” to “orchestrating,” and most programmers haven’t caught up.

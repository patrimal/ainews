OX Security: Anthropic’s MCP has an architectural RCE — Anthropic says that’s expected

Apr 16 OX Security / The Register / TechRadar

Security firm OX dropped a report Thursday detailing a design-level remote-code-execution flaw in Anthropic’s Model Context Protocol that spans every official SDK (Python, TypeScript, Java, Rust). MCP’s STDIO interface executes the command to launch a local server process before verifying the process starts — so a malicious command passed at registration runs, then returns an error. OX estimates 150M+ downloads, 7,000+ publicly reachable servers, and up to 200,000 vulnerable instances across 200+ OSS projects. They demonstrated the exploit on six live production platforms and filed 10+ Critical/High CVEs from this single root cause. Anthropic’s response: this is “expected behavior,” the STDIO execution model is a secure default, and sanitization is the developer’s responsibility. They declined to patch the protocol.

The Register’s angle: “Anthropic won’t own MCP design flaw putting 200K servers at risk.” A single manifest-only or allowlist change at the SDK level would push protection to every downstream library and user. Instead, Anthropic is treating the protocol the way OpenSSL treated certificate validation circa 2012 — technically correct, strategically negligent, and guaranteed to produce a decade of CVEs with “the developer should have known” as the official answer.

Physical Intelligence’s π0.7 folds laundry on a robot it was never trained on

Apr 16 Physical Intelligence / TechCrunch / The Decoder

Physical Intelligence released π0.7, a steerable robot foundation model that caught its own researchers off guard. The headline result: they asked π0.7 to control a bimanual UR5e to fold laundry even though the training set contained zero laundry-folding data with that robot, and it did it consistently. Across a benchmark suite — making coffee, folding laundry, assembling boxes — the generalist model matched the performance of specialists that had been fine-tuned on each task. The team is calling it the first demonstration of “compositional generalization” in robotics: recombining skills across tasks and embodiments to do things never explicitly trained. The training trick: diverse multimodal prompts that specify not just what to do but how, using data from many robots, human demos, and autonomous policy rollouts.

Physical Intelligence’s own hedging matters: they call these “early signs” and “initial demonstrations,” not a deployed product. But this is the behavior everyone has been waiting for in robotics — the thing LLMs do when you ask them to summarize a genre they’ve never seen, applied to dexterous manipulation. If it replicates, it’s the clearest signal yet that robot foundation models scale the same way language models did.
